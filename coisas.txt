O Host-Bot foi projetado em módulos onde cada um realizará sua função e passará a informação produzida para o módulo seguinte a fim de realizar a tarefa desejada.
A primeira ação a ser executada é o treinamento. Ele serve para gerar o modelo usado pelo interpretador a fim de identificar a intenção e as entidades presentes na mensagem enviada pelo usuário. Para treinar é usado um conjunto de exemplos composto por frases, onde cada frase é associada a uma intenção e a uma lista de entidades. O treinamento só precisa ser realizado novamente se o conjunto de dados usados para treinar for alterado e o modelo precisar considerar essas alterações.
A interação entre o usuário e o chatbot começa pelo módulo de comunicação que funciona como interface de diálogo entre o usuário e o chatbot utilizando mensagens de texto. Esse módulo possui duas funcionalidades: passar as informações vindas do usuário para o módulo de análise de mensagens do bot e passar a resposta gerada pelo bot para o usuário.
Após receber a mensagem, é executado o módulo de análise. Ele utiliza o modelo gerado pelo módulo de treinamento e a partir dele é feito o processamento da  mensagem enviada pelo usuário. Cada frase passada para esse módulo é interpretada de forma a serem identificadas a intenção que o usuário tem com essa mensagem e as entidades encontradas nela.
A partir das entidades e da intenção da mensagem são feitas consultas ao banco de dados para obter as informações desejadas pelo usuário e assim gerar gerar as respostas.
Os dados precisam ser extraidos de alguma fonte que contenha as informações que se deseja que chatobot use para responder as perguntas do usuário e então devem ser armazenados no banco de dados na forma de registros em tabelas.
Após o processamento da mensagem enviada pelo usuário e feitas as consultas devidas para geração da resposta, é montada a mensagem que é então enviada para o usuário por meio do módulo de comunicação.
Esse processo é repetido enquanto houver mensagens vindas do usuário.
